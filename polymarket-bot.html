CARA KERJA — 3 STRATEGI DETEKSI

1. Binary Sum Arbitrage (YES + NO ≠ $1.00)

Di market binary, harga YES + harga NO seharusnya = $1.00. Saat sum < $0.97, beli kedua sisi → profit dijamin saat resolve. Saat sum > $1.03, sebaliknya.

2. Multi-Outcome Sum (Categorical Events)

Untuk event seperti "Siapa Presiden 2028?" dengan banyak kandidat, total semua harga YES seharusnya = $1.00. Jika < $0.95, beli semua = profit dijamin saat resolve.

3. Wide Bid-Ask Spread (Market Making)

Order book dengan bid-ask gap besar. Strategi: pasang limit order di tengah, ambil spread.

REALITA YANG PERLU DIPAHAMI

• Market besar (volume jutaan dolar) jarang punya inefisiensi >1% karena banyak bot pro yang menjaga harga.

• Inefisiensi 2-5% biasanya muncul di market kecil dengan likuiditas tipis — tapi slippage akan menggigit.

• Setiap trade bayar gas Polygon (~$0.20-0.50). Pastikan profit > gas + 2% buffer.

• Strategi YES+NO arbitrage butuh modal 2x: $5 untuk YES + $5 untuk NO = $10 total terkunci sampai resolve (bisa berbulan-bulan).

• Polymarket diblokir di AS. Pastikan akses Anda legal di yurisdiksi Anda.

• Mulai dengan mode simulasi minimal 2 minggu. Jika tidak profit di simulasi, tidak akan profit di live.

SETUP UNTUK TRADING NYATA

1. Buat akun di polymarket.com

2. Deposit USDC ke wallet Polymarket (jaringan Polygon)

3. Buka clob.polymarket.com → Create API Credentials

4. Masukkan API Key/Secret/Passphrase di tab Pengaturan

5. Test koneksi → pastikan saldo tampil benar

6. Set Manajemen Risiko KECIL dulu (max trade $5)

7. Jalankan dalam mode SIMULASI selama 1-2 minggu

8. Hanya nonaktifkan mode simulasi setelah Anda paham polanya

EKSEKUSI MANUAL VS OTOMATIS

Bot ini tidak eksekusi live order langsung dari browser karena private key tidak boleh disimpan di browser (resiko keamanan parah).

Saat peluang muncul, tap kartu → Polymarket terbuka → eksekusi manual. Total waktu ~10-20 detik.

Untuk eksekusi otomatis penuh, butuh backend server Node.js. Minta Claude buat backend jika Anda butuh.
